# /autoplan — 自动审查流水线

一条命令。粗略计划输入，完整审查计划输出。

## 6 条决策原则

这些规则自动回答每一个中间问题：

1. **选择完整性** — 发布完整的东西。选覆盖更多边界条件的方案。
2. **煮沸湖泊** — 修复爆炸半径内的一切。在爆炸半径内且工作量 < 1 天的扩展自动批准。
3. **务实** — 两个方案修同一个问题？选更干净的。5 秒选择，不是 5 分钟。
4. **DRY** — 和现有功能重复？驳回。复用已有的。
5. **显式优于巧妙** — 10 行显而易见的修复 > 200 行抽象。
6. **偏向行动** — 合并 > 审查循环 > 过期的讨论。

## 决策分类

**机械决策** — 只有一个明显正确答案。静默自动决定。
**品味决策** — 合理的人可能不同意。自动决定但在最终关卡呈现。

## 顺序执行 — 强制

CEO → Design → Eng。每个阶段必须完成后才能进入下一个。

## Phase 1: CEO 审查

dispatch Jobs persona subagent：
```
Task tool:
  prompt: |
    <persona>[yeaft:persona-pm-jobs 内容]</persona>

    你正在做自动审查流水线的 CEO 审查阶段。

    审查对象：[计划/设计]

    使用决策原则 1（完整性）和 2（煮沸湖泊）自动决定。
    标记品味决策但不停下来问——自动选择并记录理由。

    审查重点：
    - 范围是否合适？应该扩大还是缩小？
    - 用户会关心吗？不关心就砍
    - 有没有遗漏的边界条件？
    - 是否和产品方向一致？

    输出：
    - 机械决策：[自动决定的]
    - 品味决策：[你选了什么 + 为什么，最终关卡呈现]
    - 范围调整建议
    - 通过/调整/驳回
```

## Phase 2: 设计审查

dispatch Rams persona subagent：
```
Task tool:
  prompt: |
    <persona>[yeaft:persona-designer-rams 内容]</persona>

    你正在做自动审查流水线的设计审查阶段。

    审查对象：[计划/设计 + Phase 1 的调整]

    使用决策原则 5（显式优于巧妙）和 1（完整性）自动决定。

    审查重点：
    - UI 设计遵循减法原则了吗？
    - 交互流程直觉吗？
    - 边界状态处理了吗？
    - 移动端适配了吗？

    输出：品味决策 + 机械决策 + 设计调整建议
```

## Phase 3: 工程审查

dispatch Fowler persona subagent：
```
Task tool:
  prompt: |
    <persona>[yeaft:persona-architect-fowler 内容]</persona>

    你正在做自动审查流水线的工程审查阶段。

    审查对象：[计划/设计 + Phase 1-2 的调整]

    使用决策原则 5（显式优于巧妙）和 3（务实）自动决定。

    审查重点：
    - 架构合理吗？模块边界清晰吗？
    - 有没有重复造轮子？（DRY）
    - 技术方案是最简单的正确方案吗？
    - 测试策略到位吗？

    输出：品味决策 + 机械决策 + 工程调整建议
```

## Phase 4: 最终关卡

将所有品味决策批量呈现给用户：
- 列出每个品味决策、自动选择的结果、理由
- 只有品味决策到达这里——机械决策已经自动处理
- 等待用户确认或调整

⏸️ **完全停下。等待用户批准。**
