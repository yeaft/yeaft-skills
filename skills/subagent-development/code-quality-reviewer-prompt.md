# Code Quality Reviewer Prompt 模板

dispatch 代码质量审查 subagent 时使用此模板。

**目的：** 验证实现质量（清晰、可测试、可维护）

**只在规格合规审查通过后才 dispatch。**

```
Task tool:
  description: "审查任务 N 的代码质量"
  prompt: |
    <persona>
    [从 yeaft:persona-dev-torvalds 加载的内容]
    </persona>

    你正在审查代码质量。

    ## 实现内容

    [从实现者的报告中]

    ## 需求/计划

    任务 N，来自 [计划文件]

    ## 变更范围

    BASE_SHA: [任务前的 commit]
    HEAD_SHA: [当前 commit]

    ## 你的工作

    审查代码质量，重点关注：

    **结构：**
    - 每个文件是否有一个清晰职责和明确接口？
    - 单元是否可以独立理解和测试？
    - 实现是否遵循计划中的文件结构？
    - 有没有创建已经很大的新文件或显著膨胀现有文件？

    **质量：**
    - 命名清晰准确吗？
    - 代码干净可维护吗？
    - 错误处理到位吗？
    - 有没有性能问题？

    **测试：**
    - 测试验证行为还是 mock？
    - 边界条件覆盖了吗？
    - 测试可读且可维护吗？

    **纪律：**
    - 遵循了 YAGNI 吗？
    - 遵循了项目既有模式吗？
    - 没有偷偷夹带不相关改动？

    报告：
    - **优点：** [做得好的地方]
    - **问题：** Critical / Important / Minor [具体列出，附文件:行号]
    - **结论：** ✅ 通过 | ❌ 需要修复
```
