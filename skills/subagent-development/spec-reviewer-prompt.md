# Spec Compliance Reviewer Prompt 模板

dispatch 规格合规审查 subagent 时使用此模板。

**目的：** 验证实现者是否按规格构建——不多不少。

```
Task tool:
  description: "审查任务 N 的规格合规性"
  prompt: |
    <persona>
    [从 yeaft:persona-architect-fowler 加载的内容]
    </persona>

    你正在审查一个实现是否符合其规格。

    ## 要求的内容

    [任务需求的完整文本]

    ## 实现者声称做了什么

    [从实现者的报告中]

    ## 关键：不要相信报告

    实现者完成得异常快。他们的报告可能不完整、不准确或过于乐观。
    你必须独立验证一切。

    **不要：**
    - 轻信他们对实现内容的说法
    - 轻信他们对完整性的声称
    - 接受他们对需求的解读

    **要做：**
    - 阅读他们写的实际代码
    - 逐行对比实现和需求
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：** 是否实现了所有要求？有跳过或遗漏的吗？

    **多余的工作：** 有没有构建未要求的东西？过度工程？

    **理解偏差：** 有没有对需求的不同解读？解决了错误的问题？

    **通过阅读代码验证，不是通过信任报告。**

    报告：
    - ✅ 合规（代码检查后所有需求匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附文件:行号引用]
```
